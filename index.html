WDYM OSINT?

Ayo listen up fam. OSINT stands for Open Source Intelligence. Its basically collecting and analyzing publicly available data from open sources to produce actionable intel. We're talkin websites, social media, news articles, public records, gov databases - anything thats out there in the open.

Originally this was military-grade stuff used alongside SIGINT (signal intel) and HUMINT (human intel) for national security and military strategy. Now its everywhere - cybersecurity pros, private investigators, journalists, and even regular folks use it daily.

The intelligence cycle goes: Plan -> Collect -> Process -> Analyze -> Share. U gotta be careful tho - verify ur sources, keep data fresh, and always stay within legal and ethical boundaries. Just cuz its public dont mean u can do whatever u want with it.

Google Dorking - The Cheat Sheet

Google Dorking (aka Google Hacking) is using advanced search operators to find sensitive info that aint supposed to be public. Hackers use it to find exposed files, login pages, and confidential data. Security teams use it to find their own leaks before the bad guys do.

Heres the full breakdown of the most powerful operators. Copy-paste and modify for ur targets.

site:target.com
What it does: Restricts all results to a specific domain or website. This is ur starting point for any target. Instead of searching the whole web, u only get results from that one site. Combine with other operators for precision.
Example: site:github.com "api_key" - finds API keys leaked on GitHub
inurl:keyword
What it does: Searches for pages where the URL contains a specific keyword. Great for finding admin panels, login pages, config files, and backend interfaces.
Example: inurl:admin.php site:target.com - finds admin login pages
Example: inurl:.env - finds exposed environment files with credentials
intitle:"index of /"
What it does: Finds open directory listings - basically folders on web servers that arent properly secured and show all their contents. These can contain backups, source code, internal docs, and sensitive files.
Example: intitle:"index of /" "config.json" - finds config files in open dirs
Risk: Misconfigured permissions exposing internal directories publicly
filetype:pdf OR filetype:docx OR filetype:xlsx
What it does: Filters results to specific file types. Organizations accidentally upload sensitive docs all the time - financial reports, internal memos, employee lists, password spreadsheets. This finds em.
Example: filetype:pdf "confidential" site:target.com
Example: filetype:xlsx "password" "email" - finds password spreadsheets
intext:"username" "password" filetype:txt
What it does: Searches for specific text within the body content of pages. When combined with filetype, it can find text files containing login credentials that someone left exposed on a server.
Example: intext:"password" filetype:txt site:target.com
Risk: Exposed credential files are a goldmine for attackers
cache:example.com
What it does: Shows Googles cached (saved) version of a webpage. Even if the original page is deleted or the site is down, Google might still have a copy. Useful for recovering deleted content or viewing sites that are currently unavailable.
Example: cache:target.com/page - view cached version
intitle:"admin login" OR intitle:"control panel"
What it does: Searches for specific text in the page title. Admin login pages, control panels, and backend interfaces often have predictable titles. Finding these is step one for brute-force or credential stuffing attacks.
Example: intitle:"phpMyAdmin" site:target.com - finds database admin panels
inurl:.git OR inurl:.svn OR inurl:.env
What it does: Finds exposed version control folders and environment files. .git folders contain the entire source code history. .env files often have API keys, database passwords, and other secrets. These should NEVER be public.
Example: inurl:.git "config" site:target.com
Risk: Source code leaks, API key exposure, complete system compromise
ext:sql "backup" OR ext:bak OR ext:old
What it does: Finds backup files and old versions of databases. Developers often leave backup files on production servers. These can contain complete database dumps with user data, passwords, and sensitive records.
Example: ext:sql "users" "password" site:target.com
intitle:"Apache2 Ubuntu Default Page"
What it does: Finds servers that still have default configurations. An Apache default page usually means the server was set up but not properly configured or secured. These are often forgotten development or staging servers full of vulnerabilities.
Risk: Unpatched systems, default credentials, exposed services
inurl:phpinfo.php
What it does: Finds PHP info pages that reveal detailed server configuration. These pages show PHP version, loaded extensions, server paths, environment variables, and system architecture. Its basically a roadmap for attacking the server.
Risk: Information disclosure leading to targeted exploits
allintitle:keyword1 keyword2
What it does: Similar to intitle but requires ALL specified keywords to appear in the page title. More restrictive than intitle which only needs one match.
Example: allintitle:confidential report 2026
related:target.com
What it does: Finds websites that Google considers similar to the target. Useful for finding competitors, affiliated sites, or alternative entry points during reconnaissance.
Example: related:facebook.com - finds similar social platforms
AROUND(5) "term1" "term2"
What it does: Finds pages where two terms appear within X words of each other. The number in parentheses controls the proximity. Great for finding associations between people, companies, or events.
Example: "CEO" AROUND(3) "fired" site:target.com
before:2024-01-01 after:2023-01-01
What it does: Time-based filters. "before" shows content indexed before a date, "after" shows content indexed after. Combine both to search within a specific time window. Useful for finding old leaks, historical data, or content from a specific period.
Example: site:pastebin.com "password" after:2026-01-01

Famous OSINT Tools in the Game

Heres the heavy hitters in the OSINT world. All open-source, all legal, all powerful.

theHarvester
Python tool for harvesting emails, subdomains, hosts, employee names, open ports, and banners from different public sources like search engines and PGP key servers.
Maltego
Visual link analysis tool that maps relationships between people, domains, IPs, and social media accounts. The graphical interface makes complex recon easy to follow.
Shodan
The search engine for the Internet of Things. Finds webcams, routers, servers, industrial control systems, and basically anything connected to the internet. Scary how much is exposed.
SpiderFoot
Automated OSINT reconnaissance platform. Integrates with over 200 data sources to gather intel on domains, IPs, emails, names, and more. Highly modular and extensible.
Recon-ng
Full-featured web reconnaissance framework written in Python. Works like Metasploit but for OSINT. Has a modular structure with tons of modules for different data sources.
OSINT Framework
A web-based collection of OSINT tools and resources organized by category. Think of it as a directory of every OSINT tool ever made. Great for discovery.
Sherlock
Hunt down social media accounts by username across 300+ social networks. Fast, lightweight, and brutally effective for username recon.
Have I Been Pwned
Check if an email or username has been compromised in data breaches. Essential for credential stuffing research and breach analysis.
